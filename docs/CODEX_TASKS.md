## 2. 增加串口自动枚举功能

```md
请基于 AGENTS.md 修改本项目，增加串口自动枚举功能。

目标：
1. 使用 QSerialPortInfo::availablePorts() 获取系统可用串口。
2. 在 Backend 暴露可供 QML 使用的端口列表，例如 QStringList availablePorts。
3. 在 ConfigPanel.qml 中把端口名输入改进为下拉选择，同时保留手动输入或刷新能力。
4. 选择端口后同步到 backend.portName。
5. 不要破坏 simulationMode。仿真模式仍然可以不依赖真实串口运行。

要求：
- 串口枚举逻辑放在 C++，不要用 QML JS 做系统查询。
- 如新增 Q_PROPERTY，需要 getter、NOTIFY 信号和刷新方法。
- 如果使用 QStringList，请补齐显式 include。
- 给出修改文件清单和测试步骤。
```

## 3. 修复仿真模式开关不明显或不生效

```md
请检查并修复本项目的 simulationMode 逻辑。

目标：
1. ConfigPanel.qml 中明确显示当前是否为仿真模式。
2. 切换 simulationMode 后，Backend 配置要更新，SerialPortWorker 要收到新配置。
3. 仿真模式下即使没有真实串口，也能产生温度/压力/液位/流量和趋势数据。
4. 真实串口模式下不要继续读取模拟数据。
5. 状态栏或通信统计中要能看出当前是 Simulation 还是 Serial。

要求：
- 保持 Backend 作为 QML 和 Worker 的桥梁。
- 不要在 QML 中直接操作 SerialPortWorker。
- 修改后说明配置从 QML 到 Backend 再到 SerialPortWorker 的链路。
```

## 4. 修复 QML 布局遮挡问题

```md
请修复 QML 页面布局遮挡问题。

现象：
- 底部按钮可能被遮挡。
- 右侧配置区显示不完整。
- 窗口缩小时部分控件不可见。

目标：
1. Main.qml 主布局在常见窗口尺寸下不遮挡。
2. ConfigPanel.qml 内容较多时可以滚动。
3. 连接、断开、确认报警等关键按钮始终可见。
4. 不要改动 C++ 业务逻辑。

要求：
- 只修改必要的 QML 文件。
- 使用 QtQuick.Layouts 的布局能力，不要硬编码大量绝对坐标。
- 给出修改文件和验证窗口尺寸。
```

## 6. 增加报警 CSV 导出

```md
请给本项目增加报警记录 CSV 导出功能。

目标：
1. 从当前报警模型或数据库导出报警记录到 CSV。
2. 导出不能阻塞 UI 线程。
3. 文件路径可由 QML 触发，必要时先使用默认目录。
4. CSV 字段包含：时间、变量名、报警值、状态文本。

要求：
- 如果从数据库导出，不能在 UI 线程直接长时间查询。
- 如果从模型导出，要考虑大表数据量。
- 可使用 QThreadPool/QtConcurrent，但要与现有 QThread Worker 架构兼容。
- 给出导出流程、修改文件和测试步骤。
```

## 7. 增强数据库写入可靠性

```md
请审查并增强 DatabaseWorker 的报警批量写入可靠性。

目标：
1. 确认 QSqlDatabase 的创建、使用、关闭、removeDatabase 都在数据库线程。
2. 检查队列溢出策略是否安全。
3. 批量 flush 使用事务。
4. shutdown 时尽量 flush 剩余数据。
5. 数据库错误通过信号返回 Backend 并显示到状态栏。

要求：
- 不要把数据库操作移动到 Backend 或 QML。
- 不要跨线程传递 QSqlDatabase/QSqlQuery。
- 修改后说明数据库生命周期和线程归属。
```

## 11. 线程安全审查任务

```md
请对当前项目做一次线程安全审查，先不要写代码。

重点检查：
1. Backend、SerialPortWorker、DatabaseWorker 的线程归属。

7. shutdown 顺序是否安全。

- 按文件列出风险点。
- 每个风险点给出修改建议。
- 区分 P0 必改、P1 建议、P2 优化。
```
